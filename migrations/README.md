# Migrations folder

## Overview

This directory holds [Alembic](https://alembic.sqlalchemy.org/) revisions for the database schema. The URL comes from `DatabaseSettings` (`src/configs/database_settings.py`) via environment variables (see `make setup` / `.env`). New models used with `--autogenerate` must be imported in `migrations/env.py` so they are registered on `Base.metadata`.

Run Alembic from the **repository root** and pass the config explicitly:

```bash
uv run alembic -c migrations/alembic.ini <command>
```

## Naming convention

Revisions created with `scripts/create_migration.sh` or `make migration` are renamed to:

`YYYY-MM-DD_HH:MM:SS_<revision_message_slug>.py`

Example: `2026-04-03_12-00-00_create_users_table.py`

The slug is derived from the revision message (spaces become underscores; only letters, digits, and underscores remain). This keeps a clear chronological order alongside Alembic’s internal revision IDs.

## Creating revisions

### Empty revision (project default)

Interactive script (also renames the file to match the convention above):

```bash
make migration
# or
bash scripts/create_migration.sh
```

This runs `alembic revision -m "<slug>"` (no autogenerate), then renames the new file under `migrations/versions/`.

### Autogenerate from models

Compares SQLAlchemy models to the **current database** and emits operations. Ensure the DB is reachable and models are imported in `env.py`.

```bash
uv run alembic -c migrations/alembic.ini revision --autogenerate -m "short_description"
```

Autogenerate does **not** apply the rename step from `create_migration.sh`. If you want the date/time filename prefix, rename the new file under `migrations/versions/` to match the convention, or create an empty revision and paste the generated operations after review.

Always review autogenerated migrations before applying them.

## Applying and inspecting migrations

| Action | Command |
|--------|---------|
| Apply all pending revisions | `make migrate` or `POSTGRES_HOST=localhost uv run alembic -c migrations/alembic.ini upgrade head` |
| Downgrade to a revision | `uv run alembic -c migrations/alembic.ini downgrade <revision>` |
| Current DB revision | `uv run alembic -c migrations/alembic.ini current` |
| History | `uv run alembic -c migrations/alembic.ini history` |
| Show one revision | `uv run alembic -c migrations/alembic.ini show <revision>` |

`make migrate` sets `POSTGRES_HOST` to `localhost` when unset (host machine talking to the published Postgres port from Compose). From another container on the Compose network, use e.g. `make migrate POSTGRES_HOST=postgres`.

## Layout

- `alembic.ini` — Alembic configuration (`script_location` is this folder).
- `env.py` — Offline/online migration context, metadata, and DB URL wiring.
- `versions/` — Revision scripts.
